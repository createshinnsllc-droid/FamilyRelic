### 1. Purpose & Scope

This SOP defines the non-destructive workflow for capturing, verifying, and escalating results from sub-agent execution. It ensures:

- **Receipt-based completion claims**: Every material claim is tagged verified vs uncertain.
- **Evidence tagging**: Prefer explicit `unknown` over guessing.
- **Delegation-first execution** for cost control, with clear escalation triggers.

### 2. Definitions

| Term | Definition |
|------|------------|
| **Orchestrator** | Main session requesting work (user's direct chat) |
| **Worker** | Spawned sub-agent executing delegated task |
| **Result Packet** | Structured output containing diagnosis, commands, verification, expected result + checklist |
| **Quality Gate** | Threshold for auto-escalation (output confidence/quality below task risk) |

### 3. Workflow Diagram

```
[Request] 
    ↓
[Spawn Worker] → runtime="acp", thread=true, mode="session"
    ↓
[Worker executes task in sandbox]
    ↓
[Orchestrator polls via heartbeat or status query]
    ↓
[Result Packet generated by Orchestrator]
    ↓  ━━━► [Verification Checklist Applied] ━━━
    ↓
[Pass → Archive / Fail → Escalate to Premium Model]
```
